Herdr is mouse-native. You can click panes, tabs, workspaces, and agents, drag split borders, and use right-click menus without learning a single keybinding. Keyboard control is an optional layer, not a requirement.

## What the prefix is

A terminal multiplexer sits between your terminal and the programs running inside it. Those programs already use most key combinations: `ctrl+c` interrupts, `ctrl+r` searches history, editors claim nearly everything else. If Herdr grabbed common keys directly, it would break the programs inside it.

The prefix solves this. Press the prefix key, default `ctrl+b`, and the next keypress goes to Herdr instead of your terminal. `prefix+c` means: press `ctrl+b`, release, then press `c`. One reserved key instead of dozens.

Press `prefix+?` at any time to see every active binding.

## Learn these five first

| Action | Key |
| --- | --- |
| New tab | `prefix+c` |
| Split right / down | `prefix+v` / `prefix+minus` |
| Move between panes | `prefix+h/j/k/l` or `Alt+h/j/k/l` |
| Workspace navigation | `prefix+w` |
| Detach, leave everything running | `prefix+q` |

These cover most daily movement. Everything else can stay on the mouse.

## The rest, by task

Panes:

| Action | Key |
| --- | --- |
| New pane (auto direction) | `Alt+n` |
| Zoom the focused pane | `prefix+z` or `Alt+z` |
| Close pane | `prefix+x` or `Alt+x` |
| Grow / shrink pane | `Alt+=` / `Alt+-` |
| Swap panes | `prefix+shift+h/j/k/l` |
| Resize mode | `prefix+r` |
| Copy mode | `prefix+[` |

Tabs:

| Action | Key |
| --- | --- |
| Next / previous tab | `prefix+n` / `prefix+p` |
| Move tab left / right | `Alt+i` / `Alt+o` |
| Jump to tab 1–9 | `prefix+1..9` |
| Rename tab | `prefix+shift+t` |
| Close tab | `prefix+shift+x` |

Workspaces and session:

| Action | Key |
| --- | --- |
| New workspace | `prefix+shift+n` |
| Rename workspace | `prefix+shift+w` |
| Close workspace | `prefix+shift+d` |
| Goto picker | `prefix+g` |
| Toggle sidebar | `prefix+b` |

The full keymap and the binding syntax live in the [keybinding reference](/docs/configuration/#keybindings).

## Copy mode

Press `prefix+[` to enter copy mode for the focused pane. Use `h/j/k/l`, `w/b/e`, and `{`/`}` to move, `v` or Space to start a selection, `y` or Enter to copy it, and `q` or Esc to leave without copying. Mouse drag-select copies without entering copy mode at all.

## Change anything

Every binding is configurable, including the prefix itself:

```toml
[keys]
prefix = "ctrl+a"
```

## Alt shortcuts

Herdr ships default `Alt+key` shortcuts that work without entering prefix mode, inspired by Zellij's ergonomics. These are active in terminal mode alongside the existing prefix bindings.

| Shortcut | Action |
| --- | --- |
| `Alt+h` / `Alt+j` / `Alt+k` / `Alt+l` | Focus pane left / down / up / right |
| `Alt+n` | New pane (auto-split: side-by-side for wide panes, stacked otherwise) |
| `Alt+x` | Close focused pane |
| `Alt+z` | Toggle zoom on focused pane |
| `Alt+=` / `Alt+-` | Grow / shrink focused pane by 5% |
| `Alt+i` / `Alt+o` | Move current tab left / right |

All existing `prefix+*` bindings continue to work. The Alt shortcuts are an additional layer, not a replacement.

:::caution[Alt+x is destructive]
`Alt+x` closes the focused pane and terminates its process immediately with no confirmation and no undo. The key is intercepted silently — the pane app never receives it. If you need `Alt+x` inside a terminal app (vim, emacs, readline), remove the Alt binding:

```toml
[keys]
close_pane = "prefix+x"
```
:::

### Known limitations

**Unconditional silent interception.** When a default Alt binding matches in terminal mode, the key never reaches the focused pane. This bites editor users: `Alt+n`, `Alt+i`, and `Alt+o` collide with vim's new-line, insert, and open-line commands, and `Alt+x` is emacs's `M-x`. A pane app expecting any of these will not receive it. The only remedy is removing the Alt binding from your config. The keystroke is also swallowed even when the action is a no-op (for example `Alt+i`/`Alt+o` with a single tab, or `Alt+=`/`Alt+-` in a single-pane tab).

**Terminal-mode-only.** Alt shortcuts fire only in terminal mode (the normal state when typing in a pane). They do not fire in any other mode — Navigate, Prefix, Resize, copy mode, or while a dialog is open.

**Outer multiplexer consumption.** If Herdr runs inside tmux, screen, or another multiplexer, that outer layer may consume the Alt key before Herdr sees it. In that case, use the prefix bindings instead.

### Diagnosing "my Alt key does nothing"

If an Alt shortcut is not working:

1. **Check your terminal.** Some terminals (especially on macOS) treat Option as a compose key that produces special characters instead of sending Alt. Enable "Use Option as Meta" or equivalent in your terminal settings.

2. **Check for an outer multiplexer.** A tmux or screen session above Herdr will consume the Alt key. Use prefix bindings instead, or configure the outer multiplexer to pass Alt through.

3. **Check your config.** If you overrode the action with a single string (e.g. `focus_pane_left = "prefix+h"`), the Alt binding is removed. Use an array to keep both: `focus_pane_left = ["prefix+h", "alt+h"]`. On upgrade, the six dual-bound actions (`focus_pane_left/down/up/right`, `close_pane`, `zoom`) gain their Alt default automatically wherever you did not set the field — only an explicit single string opts you out. If you customized any of them as a single string and want the Alt binding, switch to the array form.

4. **Alt-only actions require a modifier.** The new actions (`split_auto`, `move_tab_left`, `move_tab_right`, `resize_grow`, `resize_shrink`) have no prefix companion. Binding them to a bare printable key like `split_auto = "n"` is rejected by the unmodified-printable guard, which surfaces a startup diagnostic warning. They must use a modifier such as `alt+n`.

5. **Enable debug logging.** Set `HERDR_LOG=herdr=debug` and grep the log for `intercepted terminal direct keybinding before forwarding to pane` when you press the Alt chord. If the line appears, Herdr saw the key and acted on it; if it does not, the key was consumed before Herdr saw it (terminal, OS, or outer multiplexer).

## Going prefix-free

You can bind Herdr actions to direct chords that need no prefix at all. The hard part is knowing which chords are safe, because terminals, shells, and desktop environments already own most of the keyboard.

Any chord works as a binding: `ctrl+j`, `alt+k`, whatever fits your hands. But a chord has to survive three layers before Herdr sees it: your operating system, your outer terminal (Ghostty, iTerm2, and others ship their own defaults), and the programs running inside the pane. `ctrl+j` reaches Herdr fine, but shells and editors treat it as enter. `alt+k` is free on Linux, but macOS composes it into a special character in most terminals. If you pick chords from these families, double-check them against your own terminal and OS shortcuts.

We mapped the default keybindings of Ghostty, iTerm2, Terminal.app, kitty, WezTerm, Alacritty, Warp, Windows Terminal, GNOME Terminal, and Konsole, plus the global shortcuts of GNOME and KDE. One modifier family is almost untouched everywhere: `ctrl+alt`. Terminals leave it free, it is not affected by the macOS option-key composing behavior that blocks plain `alt` chords, and it transmits even in terminals without a modern keyboard protocol. It is the safe default recommendation; the choice stays yours.

This setup keeps the prefix bindings working and adds direct chords on top:

```toml
[keys]
focus_pane_left = ["prefix+h", "ctrl+alt+h"]
focus_pane_down = ["prefix+j", "ctrl+alt+j"]
focus_pane_up = ["prefix+k", "ctrl+alt+k"]
focus_pane_right = ["prefix+l", "ctrl+alt+l"]
previous_tab = ["prefix+p", "ctrl+alt+["]
next_tab = ["prefix+n", "ctrl+alt+]"]
new_tab = ["prefix+c", "ctrl+alt+c"]
split_vertical = ["prefix+v", "ctrl+alt+d"]
split_horizontal = ["prefix+minus", "ctrl+alt+shift+d"]
zoom = ["prefix+z", "ctrl+alt+z"]
```

A few `ctrl+alt` chords are taken elsewhere. Avoid these:

| Chord | Owned by |
| --- | --- |
| `ctrl+alt+arrows` | GNOME workspace switching, Ghostty and Konsole defaults |
| `ctrl+alt+t` | "Launch terminal" on Ubuntu and Fedora |
| `ctrl+alt+l` / `ctrl+alt+a` | KDE lock screen / attention window |
| `ctrl+alt+s` / `ctrl+alt+u` | Konsole |
| `ctrl+alt+f1..f12` | Linux virtual console switching |

If a direct chord does nothing, your terminal or desktop environment consumed it before Herdr could see it. Rebind either side: free the chord in the terminal's settings, or pick another chord in Herdr.
